QA & ANS

Javascript হচ্ছে Interpreted and object-oriented language । javascript কে web development এর মাতৃ ভাষা বলা হয় । আরেক টা বিষয় হচ্ছে javascript একটি হাই লেভেল ভাষা । এর জন্য এটা শিখা টা তুলনামুলক সহজ এর জন্য web development এর খেতরে এটা খুব জনপ্রিয় । javascript সব থেকে মজার বিষয় তা হচ্ছে অন্যান্য Programming language যেমন পুরা কোড কে Read করে তার পর output show করে কিন্তু javascript লাইন বাই লাইন পড়ে এবং output show করতে পারে এর জন্য web developer এর ক্ষেত্রে bug,error handle করতে ওনেক সাহায্য করে । javascript যখন কোড রান করে তখন web browser এর engine সাহায্য করে । সব থেকে জনপ্রিয় engine হচ্ছে chorme browser এর v8 engine । javascript যেহেতু high level language তাই সরাসরি machine language এর সাথে relation করতে পারে না তাই web browser এর engine রিলেশন করতে সাহায্য করে ।এতে করে code compailer করতে সহজ হয় ।

javascript event loop যা করে তা সহজ করে বলার চেষ্টা করছি । আমরা যখন javascript কোড লিখি তখন আমরা web browser বা কোড ইডিটর এ রান করে থাকি । কোড টা রান করার সময় web browser এ engine এর মাধ্যমে machine language এ রুপান্তর করে । দরে নেই আমরা chorme browser এর v8 engine দিয়ে রান করছি । আর v8 engine দুইটা পার্ট হচ্ছে Heap ও আরেকটা হচ্ছে Call Stack আর Call Stack (lifo=last in fast out) হিসেবে কাজ করে । Call Stack টা হচ্ছে javascript কোড এ যখন কনো function কে Call করি তখন Call Stack এসে জমতে থাকে এবং synchronous ভাবে দিতে থাকে । আর কিছু asynchronous রয়েছে যেমনঃ setTimeout function কে Call করি তখন সেটা কে সরিয়ে রাখে বলে বাবু একটু পর তুমাকে Call করব মানে waiting রাখে এখন প্রশ্ন টা হল রাখে কথায় রাখে হচ্ছে web api / c++ api তে ।রাখার পর web api সেই function কে পাটিয়ে দেয় হচ্ছে Callback Queue এর কাছে আর Callback Queue (fifo= fast in fast out ) কাজ করে থাকে । আমরা এতো খন পরজন্নতো জাই করি না কেন Call Stack কিন্ত এখনো Call করে নাই Call না করলে কিন্ত কনো কাজ করবে না তাই Call Stack এ Call করার কাজ টা করে হচ্ছে event loop ।
